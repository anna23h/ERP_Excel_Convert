## 每日流程（两阶段）

```
导出当日 ERP + 天猫全量
        │
        ▼
[阶段一] build_excel.py ──► 拣货表+面单.xlsx ──► 打印给仓库
        │                                         │
        │                              仓库捡货、纸面标无货
        │                                         ▼
        │                      办公室在「无货勾选」页把无货行填 1，另存为返回文件
        ▼
[阶段二] stage2.py（吃返回文件）──► B / C / D / 缺货记录
```

## 阶段一：生成「拣货表 + 面单」

把当日 ERP 导出和天猫全量导出放进 `raw_data/`，然后：

```bash
python3 build_excel.py <当日ERP导出> <当日天猫全量导出>
```

示例：

```bash
python3 build_excel.py raw_data/测试0611erp导出.xlsx raw_data/天猫测试.xlsx
```

- **第二个参数是「完整天猫导出」**（全量、可跨多日，唯一天猫输入）。发货范围由程序二段式自动算出：`履约状态∈{新订单,商家已接单} ∧ 面单申请状态=已完成`，无需再单独导「面单已完成名单」。
- 一次产出（均存到 `output/`，按店 VO/GW 各一份）：
  | 文件 | 用途 |
  |---|---|
  | `YYYY年MM月DD日{店}{n}单 拣货表+面单.xlsx` | 三 sheet：`拣货表`/`面单`(打印给仓库)/`无货勾选`(填 1 标无货) |
  | `今日预计发货总获单清单{店}.xlsx` | 今日要发全集的系统履约单号(新订单+商家已接单∧面单已完成)，去天猫批量获单 |
  | `新订单获单清单{店}.xlsx` | 仅 `新订单` 部分的履约单号，去天猫确认接单 |
  | `回传ERP销售上传表{店}.xlsx` | 取消/无运单/已补运单三类 Terms 写回，回传 ERP |
  | `已补运单清单{店}.xlsx` | 昨日无运单今日补出的履约单号，去天猫打面单 |
- 打印格式已设：拣货表纵向压一页宽、面单横向、页脚「第 N 页，共 M 页」。

> ERP 导出格式 **xlsx 或 csv 都可**，脚本输出一致。

## 阶段二：生成 B / C / D / 缺货记录

仓库返回、办公室在 `无货勾选` 页标完无货并**另存**后（如 `raw_data/拣货表+面单返回.xlsx`）：

```bash
python3 stage2.py --mmdd <当日MMDD> \
    --erp <当日ERP导出> \
    --tmall-full <当日完整天猫导出> \
    --nogoods <仓库返回文件>
```

完整示例：

```bash
python3 stage2.py --mmdd 0611 \
    --erp raw_data/测试0611erp导出.xlsx \
    --tmall-full raw_data/天猫测试.xlsx \
    --nogoods "raw_data/拣货表+面单返回.xlsx"
```

> 账单上传默认从 ERP 导出的 `External ID` 列直接生成，无需 `--billing`；仅当订单导出缺 ID 列时才回落用 `--billing <账单模板导出>`。

输出（存到 `output/`）：

| 文件 | 内容 |
|---|---|
| `系统履约单号.xlsx` | 实际发货订单的系统履约单号 → 上传天猫确认发货 |
| `发货表.xlsx` | 发货订单，按 GW / VO 分 sheet |
| `账单上传.xlsx` | 开账单上传表（Terms 加 `账单MMDD` 标签）→ 回传 ERP。**前提：订单导出含 `External ID` 列**（在 Odoo 订单导出模板里勾上即可），否则需用 `--billing` 喂账单模板导出 |
| `缺货记录.xlsx` | 明细（按 SKU 合并）+ SKU 汇总，供补货 |

### 参数说明

| 参数 | 必填 | 说明 |
|---|---|---|
| `--mmdd` | ✅ | 标签日期，如 `0611`，用于 D 的 `账单MMDD` 标签 |
| `--erp` | 否 | 当日 ERP 导出，默认 `raw_data/测试0611erp导出.xlsx` |
| `--tmall-full` | 否 | 当日完整天猫导出（唯一天猫输入，用于算拣货候选/缺货记录）。无货入口算候选时必需 |
| `--nogoods` | 否 | 仓库返回文件（带「无货勾选」页）。不传 = 全部发货，且不出缺货记录 |
| `--shipped` | 否 | 有货入口（白名单）：真实发货订单清单，和 `--nogoods` 二选一 |
| `--billing` | 否 | **回落用**。若订单导出(`--erp`)已含 `External ID` 列，账单上传会**直接从订单导出生成**，无需此项。仅当订单导出没有 ID 列时，才用它喂单独的账单模板导出 |
| `--outdir` | 否 | 输出目录，默认 `output` |

## 注意事项

- **真实数据**放在 `raw_data/`，已 gitignore，不进版本库、不进 iCloud。输出在 `output/`。
- 无货勾选：在 Excel 里把无货行的「无货(1=缺货)」列填 `1`（0 = 正常）。多品订单缺某件只标那一行。
- `账单MMDD` 标签统一英文数字；脚本会自动剥离旧标签再贴当日标签。
- 页边距单位为英寸（0.9 / 0.8）。
- 出库单 E 已实现：阶段二传「出库原始数据」(stock picking 全量) 即按发货订单过滤、拆 VO/GW 出 `出库单{店}.xlsx`。
