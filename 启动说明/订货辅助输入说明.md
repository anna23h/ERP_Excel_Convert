# 订货辅助（Reorder Helper）输入与使用说明

产出一张**一行一品的订货决策表**：每个待订货品的最近采购价 / 供应商 / 数量 / 当前库存 / 需补货数。
本文说明三个输入入口的**传入顺序、每个入口必须包含的字段**，以及运行方法。

---

## 输入顺序总览

| 顺序 | 输入 | 必填？ | CLI 传法 | GUI 输入行 |
|---|---|---|---|---|
| 1 | 商品/需求清单 | **必填** | 第 1 个位置参数 | Product / demand list |
| 2 | purchase order 导出 | **必填** | 第 2 个位置参数 | Purchase order export |
| 3 | 输出文件 | 选填 | 第 3 个位置参数 | Output file (optional) |
| — | product.product 主数据 | 选填 | `--master <文件>` | Product master (optional) |

---

## 入口 1：商品/需求清单（必填）

对格式**宽容**：不认死 sheet 名和列头，按值形态自动识别。接受三类文件：

| 文件类型 | 说明 |
|---|---|
| 待发货明细表 | 每周收到的全量表（优先读 `需求汇总-HK` 分表） |
| ERP 导出 | `sale.order` 导出、销售分析（PZN 嵌在 `[前缀_PZN] 名称` 里） |
| 纯 PZN 清单 | 哪怕只有一列 PZN 也能跑 |

**必须满足（二者至少其一，缺则报错「未找到含 PZN 的列」）：**

- 一列 **PZN 形态的值**。可识别的写法：
  - `前缀_PZN`：`Mucosolvan_02807988`、`[DOPH_12351236x2]`（可带件装后缀）
  - 显式 PZN 码：`PZN-02807988`
  - 整格 7~8 位数字：`02807988`（7 位自动补零）
  - 金额（带小数）、12 位后端 id、13 位 EAN **不会**被误判成 PZN
- 或一列 **产品 ID**（仅 ERP 导出有）：表头须含 `Product/ID` 或 `Product/External ID`
  （如 `Order Lines/Product/ID`）。⚠️ 裸的 `ID` / `External ID` 列**不算**——那是订单自身的 id。

**选填字段（有则用，无则对应产出列自动省略）：**

| 字段 | 表头识别方式 | 产出去向 |
|---|---|---|
| 商品名称 | 精确「商品名称」，或宽松含「名称/品名」 | Name |
| 总需求 | 精确「总需求」，或宽松含「需求」 | 总需求、Reorder Qty(=总需求−库存) |
| 平台裸价 | 待发货明细表自带分表 `采购订单-健康` / `采购订单-直营` / `健康-保税`，须有 `条形码` + `平台裸价` 两列 | 平台裸价、Price Diff(=裸价−采购价) |

---

## 入口 2：purchase order 导出（必填）

Odoo `purchase.order` **行式导出**（订单头字段只出现在每单首行，脚本自动向下填充）。

**必须包含的列（表头须逐字一致，缺任何一列直接报错）：**

| 列名 | 用途 |
|---|---|
| `Order Reference` | PO 单号（近期采购记录里带出，便于回 ERP 查） |
| `Vendor` | 供应商（自动转简称，如 PHOENIX / GEHE / AEP） |
| `Order Lines/Product/Internal Reference` | PZN 回退匹配键（从 `品牌_PZN` 里抽取） |
| `Order Lines/Unit Price` | 最近采购单价、差价计算 |
| `Order Lines/Total Quantity` | 最近采购数量 |

**强烈建议勾选（选填，但直接影响质量）：**

| 列名 | 缺失的后果 |
|---|---|
| `Order Lines/Product/ID`（或 `…/External ID`） | 没有就只能全走 PZN 匹配——官方 PZN 有空白/脏值、IntRef 可能嵌旧 PZN，ID 匹配可绕开这些坑 |
| 日期列，三选一：`Order Lines/Order Date` / `Order Lines/Created on` / `Order Lines/Order Deadline` | 无法按时间排序，「最近采购」可能取错笔 |
| `Product/Quantity On Hand` | 当前库存列空（除非传了 master） |

> 供应商名含 `alibaba` 的行会整行剔除（那是我方销售平台伪装的客户单，非供应商）。

---

## 入口 3：product.product 主数据（选填，`--master`）

产品主数据导出。作用：①用**干净**的官方字段替换从名称里抠出的身份信息；②桥接「PZN 已更新、但名称/Internal Reference 还嵌旧 PZN」的错位；③让外部 PZN 输入也能走 **PZN → master → ID → PO** 的 ID 匹配。

**传了就必须包含的列（缺则报错）：**

| 列名 | 用途 |
|---|---|
| `Internal Reference` | 与 PO 侧桥接（嵌入 PZN 双键索引） |
| `PZN` | 官方 PZN（产出展示优先用它） |
| `Name` | 干净商品名 |

**选填列：**

| 列名 | 用途 |
|---|---|
| `Barcode` | 13 位 EAN，产出 Barcode 列 |
| `Quantity On Hand` | 当前库存（优先级高于 PO 侧库存） |
| `product/ID`（或 `ID` / `External ID`） | ERP 权威主键 → 产出 Product ID 列 + ID 匹配桥 |

---

## 匹配逻辑（逐行决定，自动）

1. 该行能拿到产品 ID（需求清单自带，或经 master 桥出）、且 PO 导出带 ID 列 → **按 ID 匹配**。
   数字形态 `200392` 与字符串形态 `__export__.product_product_200392_…` 自动归一互通。
2. 否则 → **按 PZN 匹配**（与旧版完全一致）。

| 输入组合 | 实际走的路径 |
|---|---|
| 待发货明细表 × PO | 全走 PZN（行为与旧版零变化） |
| 待发货明细表 × PO + master | PZN → master → ID → PO 三段桥，回退 PZN |
| ERP 导出 × 带 ID 的 PO | ID 直连，缺 ID 的行回退 PZN |

---

## 使用方法

**GUI（推荐，全英文界面）**：双击 `Reorder-Windows.bat`（Windows）或 `Reorder-Mac.command`（Mac），首次运行自动建环境。依次选文件 → Generate → 完成后点 Open output folder。

**CLI**：

```
python3 reorder_helper.py <需求清单.xlsx> <purchase order.xlsx> [输出.xlsx] [--master product.product.xlsx]
```

**产出**：默认写到 `output/当天日期/订货辅助-当天日期.xlsx`，最多 15 列：
`Product ID · PZN · Name · Barcode · Internal Reference · 总需求 · Quantity On Hand · Reorder Qty · 平台裸价 · Last Unit Price · Price Diff · Last Vendor · Last Qty · Last Order Date · Recent Purchases`（最近 5 笔，格式 `月-日|PO号|供应商|数量|@单价`）。
整批无值的列自动省略；查无采购记录的货品 Last Vendor 标 `No purchase record`。
订货状态 / MHD 仍人工填写。

## Quick Reference (English)

**Inputs, in order:**

1. **Product / demand list** (required) — any sheet with a PZN-like column (`Brand_02807988`, `PZN-02807988`, or a bare 7–8 digit number), or an ERP export carrying a `…Product/ID` / `…Product/External ID` column. Optional columns: name, demand qty.
2. **Purchase order export** (required) — Odoo `purchase.order` list export. **Must include exactly:** `Order Reference`, `Vendor`, `Order Lines/Product/Internal Reference`, `Order Lines/Unit Price`, `Order Lines/Total Quantity`. **Strongly recommended:** `Order Lines/Product/ID`, one date column (`Order Lines/Order Date` or `…/Created on`), `Product/Quantity On Hand`.
3. **Product master** (optional) — `product.product` export with `Internal Reference`, `PZN`, `Name` (required if provided); plus `Barcode`, `Quantity On Hand`, `product/ID` (optional).

**Run:** double-click `Reorder-Windows.bat` / `Reorder-Mac.command`, pick the files, hit *Generate*. Output lands in `output/<today>/`.

**Matching:** per row, Product ID is used when available on both sides; otherwise falls back to PZN. `No purchase record` means the item was not found in the purchase order export.
