## 决策
- 第三种"发货后取消"**保持现状不并不报**，只加护栏报警——避免引入复杂度。
- stage2 其实已与 stage1 **解耦**：有货入口不依赖"面单已完成名单"，GUI 该栏留空即可独立跑。手上有有货 list 时一律用有货入口 + 留空。
- 黑名单漏返回某店 → 该店默认全发（语义是"未标无货即发货"）。**不加护栏，用人为规则规避**（宁简勿繁，见记忆 feedback_simplicity_over_automation）。
- ERP 两份导出**不合并**：sale.order（External ID，行级）vs stock.picking（ID，单据级），连接键 Order Reference ↔ Source Document；合并会丢另一方主键、按粒度翻行/丢出库单原生列。能省的只是 ERP 销售份阶段一二共用导一次。
