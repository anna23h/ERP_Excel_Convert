## 改了什么
- 缺货记录表 `build_shortage` → 两 sheet：明细（按 SKU 排序合并相同 SKU 的 SKU 级列 + 留备注空列）+ SKU 汇总（按缺货量降序）。字段回连 ERP 增强 Barcode / 货位 / 系统在售库存。
- 办公室员工交付方案定 A（exe + GUI）：`build_excel` / `stage2` 重构出 `build()` / `run()` 供 GUI 进程内调用；`gui.py` tkinter 界面（后台线程不卡死，缺文件/异常弹窗）；`requirements.txt`、`build_exe.bat`、`启动说明/打包Windows-exe.md`。
- 账单上传优化：查明"为何没直接出可上传账单"非 bug——账单导入需 Odoo **External ID** 作匹配键，该 ID 仅在导出文件里。改 `build_D` → `build_billing`，订单导出含 ID 则直接生成账单，否则回落 `--billing` 并提示加列。
- 修 External ID 识别：列名实为"External ID"非"ID"，`find_id_col` 兼容 External ID / ID / id / 外部ID，保留来源列原名回传 Odoo。
- Windows 打包健壮化（解决 pip 安装失败）+ 新增 macOS 双击运行 `.command`。

## 决策
- 办公室交付走 **exe + GUI，全本地不上云**（数据合规）；用户自己用 Windows 机打包。
- 账单来源改为**订单导出加 External ID 列**，而非另传账单模板。
